## 🔤 重要英文詞語講解（先練這些再解題）

* **variable**：變數（題目在問「哪個變數」）
* **contains**：包含（暗示這個東西「裡面存了一些東西」）
* **a list of**：一個清單/列表（關鍵！不是單一路徑、不是函式、不是模組）
* **directories**：目錄（資料夾路徑）
* **to be searched**：會被拿去搜尋（被動語態，表示「系統會用它來查找」）
* **locate**：定位、找到（比 find 更偏「找到正確位置」）
* **required modules**：需要的模組（你 import 的東西）
* **search path**：搜尋路徑（專有名詞，常考）

✅ **題目訊號很明確：**「list of directories」＋「locate modules」＝ **Import 的搜尋清單**。

## 🧩 Databricks / Spark 前提資訊（正解說明之前先補底層概念）

在 Databricks 裡，你跑的是 **Python（通常是 CPython）+ Spark（PySpark）** 的組合：

* **Spark** 負責分散式運算，但 **Python 的 import 規則仍然是 Python 自己的機制**。
* 當你在 Notebook / Job 裡 `import xxx`，Python 會依照「模組搜尋順序」去找：

  * 你程式所在位置 / 目前工作目錄（情境不同會變）
  * Python 環境的 `site-packages`
  * 你額外加進來的路徑（例如你自己 append）
* 所以在 Databricks 遇到 `ModuleNotFoundError`，第一個直覺檢查點常常就是 **sys.path 裡到底有哪些路徑**（你的程式在哪、lib 裝在哪、Repos 掛哪裡）。

## ✅ 正解說明

### 為什麼 B 是正確的？

**技術原理：**
`sys.path` 是 Python 的 **模組搜尋路徑清單（list）**。當你執行 `import` 時，Python 會依序掃過 `sys.path` 裡的每個目錄，嘗試找到對應的 module/package。

**題目對應：**

* 「contains a list」→ `sys.path` 本質就是 list
* 「directories to be searched」→ 每個元素通常是 directory path
* 「locate required modules」→ import 系統就靠它找模組

**實務應用（Databricks 常見除錯）：**

* 印出 `sys.path` 檢查你的 module 是否在搜尋路徑中
* 必要時臨時加入路徑（但正式做法通常會用 wheel / init script / cluster library 管理）

```python
import sys

for p in sys.path:
    print(p)

# 臨時加入自訂路徑（除錯或臨時用途）
sys.path.append("/Workspace/Repos/my_project/utils")
```

## ❌ 錯誤選項排除

### 選項 A - `importlib.resource_path`

**錯誤原因:** 名稱看起來像真的，但不符合題意（而且通常不是「搜尋路徑清單」的概念）
**關鍵點:** 題目要的是「list of directories」的變數，不是資源讀取相關工具。

---

### 選項 C - `os.path`

**錯誤原因:** 概念混淆
**詳細分析：**

* `os.path` 是一個「處理路徑的工具模組」，提供很多函式（join / exists / dirname…）
* 它不是「一個存放目錄清單的變數」

✅ 一句話記住：

* `os.path`：**用來“操作”路徑字串**（工具箱）
* `sys.path`：**用來“存放”搜尋路徑清單**（清單本體）

---

### 選項 D - `pypi.path`

**錯誤原因:** 虛構名稱 / 不存在於標準 import 機制中
**補充:** PyPI 是套件索引站，不是 Python runtime 裡的一個 path 變數。

---

### 選項 E - `pyllib.source`

**錯誤原因:** 虛構名稱（標準庫沒有這個）

## 🧠 記憶法與解題技巧

### 記憶口訣

* **sys.path** = Python 會去找模組的 **WHERE list**
* **os.path** = 你用來處理路徑的 **HOW tools**

### 解題步驟

1. 看到 **list of directories** → 先鎖定「清單型變數」
2. 看到 **locate modules** → 鎖定 import 機制
3. 選項中只有 `sys.path` 同時滿足「list + import 搜尋」

### 常見陷阱警示

⚠️ 只看到 `path` 就選 `os.path`。
題目問的是 **contains a list**（內容是清單），不是「提供路徑操作功能」。
