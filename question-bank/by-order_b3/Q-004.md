<!-- ⚠️ 保持英文原文，不要翻譯 -->
A team consisting of multiple data analysts wants to work on an analytics project that involves performing basic data exploration, querying small datasets, and running analyses using Python and SQL. They ask a data engineer to configure interactive clusters to support their workloads.

Which of the following cluster access modes should the engineer configure to best support this use case?

### 選項
<!-- ⚠️ 保持英文原文，不要翻譯 -->
- **A.** SINGLE USER
- **B.** DEDICATED
- **C.** STANDARD
- **D.** NO_ISOLATION_SHARED

# 題目解析

---

## ✅ 正解說明
### 為什麼 C 是正確答案？

**STANDARD** 模式專為多人協作的一般分析工作設計。

#### Cluster Access Mode 比較
| 模式 | 多用戶 | 隔離性 | 適用場景 |
|------|--------|--------|----------|
| **STANDARD** | ✅ | 用戶隔離 | 團隊協作分析 |
| SINGLE USER | ❌ | 單人專用 | 個人開發 |
| DEDICATED | ✅ | 群組隔離 | 特殊工作負載 |
| NO_ISOLATION_SHARED | ✅ | 無隔離 | 不建議（安全風險） |

#### STANDARD 模式的特性
```
STANDARD 模式特性：
├── 多用戶共享叢集資源
├── 用戶工作負載相互隔離
├── 支援 Python 和 SQL
├── 支援 Unity Catalog
├── 成本效益高（共享資源）
└── 適合一般分析工作
```

#### 為什麼適合本題場景？
```
題目需求：
├── 多個資料分析師 → 需要多用戶支援
├── 基本資料探索 → 不需要特殊功能
├── 查詢小型資料集 → 標準效能即可
├── Python 和 SQL → STANDARD 支援
└── 團隊協作 → 需要隔離但可共享

STANDARD 滿足所有需求 ✅
```

---

## ❌ 錯誤選項排除

### A. SINGLE USER
**為什麼錯誤？**
- **只能單人使用**
- 不適合團隊協作場景
- 每個人需要自己的叢集，成本高

### B. DEDICATED
**為什麼錯誤？**
- 用於**特殊工作負載**（如 R、MLlib、RDD）
- 對於基本 Python/SQL 分析是過度設計
- 成本較高，不需要額外功能

### D. NO_ISOLATION_SHARED
**為什麼錯誤？**
- **無用戶隔離**，有安全風險
- 一般不建議用於多用戶環境
- 用戶可能互相影響

## 🧠 記憶法
### 口訣
**「Standard 標配多人用，安全隔離又省錢」**

### 選擇決策樹
```
叢集模式選擇：
│
├── 只有一個人用？
│   └── SINGLE USER
│
├── 多人協作？
│   ├── 一般 Python/SQL 分析？
│   │   └── STANDARD ← 本題答案
│   │
│   └── 需要 R/MLlib/RDD？
│       └── DEDICATED
│
└── 完全信任所有用戶（不建議）？
    └── NO_ISOLATION_SHARED
```

### Access Mode 詳細比較
| 特性 | SINGLE USER | STANDARD | DEDICATED | NO_ISOLATION |
|------|-------------|----------|-----------|--------------|
| 多用戶 | ❌ | ✅ | ✅ | ✅ |
| 用戶隔離 | N/A | ✅ | ✅ | ❌ |
| Unity Catalog | ✅ | ✅ | ✅ | ❌ |
| Python/SQL | ✅ | ✅ | ✅ | ✅ |
| R/MLlib | ✅ | ❌ | ✅ | ✅ |
| 成本 | 高（每人一個） | 低（共享） | 中 | 低 |

### 注意事項
```
STANDARD 限制：
├── 不支援 R 語言
├── 不支援 MLlib
├── 不支援 RDD API
└── 不支援某些進階功能

需要上述功能？→ 使用 DEDICATED
```
